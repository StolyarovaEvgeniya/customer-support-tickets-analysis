# Customer Support Tickets Analysis

Operational efficiency case study: testing whether ticket priority (`Urgent` vs `Low`) affects resolution time, customer satisfaction, and escalation rate.

## 🔍 What was done
- Loaded and explored 200k support tickets (Python, pandas, Google Colab)
- Filtered completed tickets (`Resolved`/`Closed`) for accurate time metrics
- Tested hypothesis: *"Do Urgent tickets get faster service?"*
- Checked secondary metrics: `customer_satisfaction_score`, `escalated` rate
- Formulated business-ready conclusions and stakeholder response template

## 📁 Files
| File | Description |
|------|-------------|
| `notebook.ipynb` | Interactive analysis with code, tables, and conclusions |

## 🛠️ Tools
- Python • pandas • Google Colab • Markdown

## 💡 Key insight
Priority label did not correlate with faster resolution, higher satisfaction, or lower escalation rate — suggesting it may be a metadata tag rather than an operational routing signal.
